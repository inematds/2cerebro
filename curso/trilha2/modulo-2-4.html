❓ Como o LLM Consulta o Wiki

Ao receber uma pergunta, o LLM não varre todos os arquivos — ele usa o index.md como mapa para identificar notas relevantes e então as lê. Isso reduz o uso de tokens drasticamente.

💬 Tipos de Perguntas Eficazes

Nem toda pergunta é igual. Perguntas específicas e contextualizadas produzem respostas muito melhores do que perguntas genéricas.

✅ Perguntas Eficazes

Factual sobre projeto:

"Qual o status atual do projeto Alpha e quais são os bloqueadores?"

Comparação:

"Quais são as diferenças entre FastAPI e Flask mencionadas no vault?"

Síntese temporal:

"Quais decisões técnicas tomamos em março de 2024?"

Relacional:

"O que o João está liderando atualmente?"

❌ Perguntas Ineficazes

Muito genérica:

"O que você sabe?"

LLM não sabe por onde começar

Fora do vault:

"Qual é a melhor linguagem de programação?"

Responde com treinamento geral, não seu vault

Sem âncora:

"Me fale sobre machine learning"

Não especifica se quer do vault ou treinamento

Impossível de responder:

"O que vai acontecer com o projeto?"

LLM não prevê o futuro — só o que foi registrado

📊 Consultando com Dataview

Para consultas estruturadas, o plugin Dataview permite queries tipo SQL diretamente no Obsidian — sem precisar do LLM. Ideal para dashboards e relatórios recorrentes.

📋 Projetos Ativos
```dataview
TABLE status, updated
FROM "wiki/projetos"
WHERE status = "active"
SORT updated DESC
```

Lista todos os projetos ativos com data de última atualização

👥 Pessoas por Projeto
```dataview
TABLE file.outlinks
FROM "wiki/pessoas"
WHERE length(file.outlinks) > 2
SORT file.name ASC
```

Lista pessoas com mais de 2 conexões no vault

⚠️ Notas Desatualizadas
```dataview
LIST
FROM "wiki/"
WHERE updated < date(today) - dur(30 days)
AND status = "active"
```

Notas ativas não atualizadas há mais de 30 dias

🔗 Notas Órfãs
```dataview
LIST
FROM "wiki/"
WHERE length(file.inlinks) = 0
AND length(file.outlinks) = 0
```

Notas sem nenhum link — candidatas à conexão ou remoção

🔧 Auditoria: Lint do Vault

A auditoria identifica problemas de saúde no vault: notas órfãs, inconsistências, links quebrados e conteúdo desatualizado. Faça semanalmente.

🔧 Processo de Auditoria

Prompt de Auditoria Semanal

Faça uma auditoria completa do vault:

1. Liste notas wiki/ sem nenhum [[link]] (órfãos)
2. Liste notas com status "active" não atualizadas
   há mais de 30 dias
3. Identifique contradições entre notas
   (mesmo conceito com informações diferentes)
4. Verifique se index.md está completo
5. Reporte links [[quebrados]] (notas referenciadas
   que não existem)

Gere um relatório em formato de checklist com
ações recomendadas para cada problema.

🎯 Casos de Uso: Quando Usar Cada Modo

O sistema tem três modos principais: consulta via LLM, consulta via Dataview e auditoria. Saber quando usar cada um economiza tempo e tokens.

Situação Melhor Abordagem Exemplo
Pergunta de contexto LLM + wiki/ "O que decidimos sobre X?"
Listar projetos ativos Dataview query TABLE status FROM "wiki/projetos"
Saúde do grafo LLM audit Prompt de auditoria semanal
Conexões rápidas Graph View (Obsidian) Visual do grafo no UI
Síntese criativa LLM + múltiplas notas "Conecte os conceitos de X e Y"

✓Cada modo tem seu caso de uso ideal
