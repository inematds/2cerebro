🔄 O que é "Ingerir"

Ingerir é o processo de transformar captura bruta em conhecimento estruturado. O LLM lê um arquivo de raw/, extrai os conceitos e cria ou atualiza notas no wiki/ com links, frontmatter e estrutura.

📝 O Prompt de Ingestão

O prompt de ingestão é o instrumento principal. Ele deve ser específico sobre o arquivo a processar e referenciar o CLAUDE.md para que o LLM saiba as regras.

Prompt Padrão de Ingestão

# Prompt básico
Ingira o arquivo raw/reunioes/2024-03-20.md
seguindo as regras do CLAUDE.md.

# Prompt completo com instruções
Processe o arquivo raw/reunioes/2024-03-20.md:

1. Leia o arquivo completamente
2. Identifique todos os conceitos, projetos e pessoas mencionados
3. Para cada um:
   - Crie ou atualize a nota wiki/ correspondente
   - Adicione [[links]] para notas relacionadas já existentes
   - Mantenha o frontmatter atualizado (updated, status)
4. Atualize wiki/index.md se criar notas novas
5. Registre todas as ações em wiki/log.md

Não modifique o arquivo raw/ original.
✅ Faça
  • → Especifique o arquivo exato
  • → Mencione "seguindo CLAUDE.md"
  • → Peça atualização do log.md
  • → Processe um arquivo por vez
❌ Evite
  • → Processar 10 arquivos de uma vez
  • → Permitir edição do raw/
  • → Omitir atualização do index.md
  • → Esquecer de revisar o resultado

📄 Anatomia de uma Nota Wiki

Uma boa nota wiki tem cinco seções padrão. O LLM gera essas seções automaticamente durante a ingestão — mas você precisa entendê-las para revisar o resultado.

🔍 Exemplo Completo: Reunião → Wiki

Veja um exemplo real completo: uma anotação bruta de reunião transformada em notas wiki estruturadas pelo LLM.

📥 raw/ — Antes
standup 20/03
- alpha atrasado por bug no pg
- joao vai resolver hoje
- demo sexta pra cliente
- discutimos mudar pra fastapi
  (flask ta lento demais)
- meeting com vendas na quinta
  perguntou sobre multi-tenant
  ainda nao temos isso
📤 wiki/ — Depois
--- projeto-alpha.md ---
status: at-risk
blocker: bug PostgreSQL
demo: 2024-03-22 (cliente)
[[joao]] responsável bugfix
[[fastapi]] migração planejada

--- fastapi.md (criado) ---
Alternativa ao Flask considerada
em 2024-03-20. Motivação: perf.
[[flask]] [[projeto-alpha]]

--- multi-tenant.md (criado) ---
Feature solicitada por vendas
em 2024-03-20. Status: backlog.
[[projeto-alpha]] [[vendas]]
💡 O que o LLM fez automaticamente
  • → Identificou 3 conceitos distintos em uma nota bagunçada
  • → Criou 2 notas novas e atualizou 1 existente
  • → Adicionou datas relativas ("20/03") como datas absolutas
  • → Criou links cruzados entre todas as notas novas

⏱️ Frequência e Rotina de Ingestão

A ingestão funciona melhor com ciclos regulares e curtos. Não acumule semanas de raw/ antes de processar — o ideal é ingerir no mesmo dia ou no dia seguinte à captura.

📅 Ritmo de Ingestão Recomendado

6

🛡️ Revisando e Validando Resultados

Sempre revise o que o LLM criou. O LLM é bom em estruturar, mas você é responsável pela qualidade e veracidade do conteúdo. A revisão leva 2-3 minutos por ingestão.

Checklist de Revisão
Sinais de Problema
  • ⚠LLM inventou informação que não estava no raw/
  • ⚠Links para notas inexistentes
  • ⚠raw/ foi modificado
  • ⚠Nota criada com nome duplicado
✅ Conceitos-chave do módulo
✓Ingerir = raw/ → estrutura wiki/ com links
✓Prompt especifica arquivo + menciona CLAUDE.md
✓5 seções padrão por nota wiki
✓1 raw/ gera N notas wiki especializadas
✓Ciclo diário de ingestão + review semanal
✓Sempre revisar antes de confiar no resultado
