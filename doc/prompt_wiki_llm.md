# Prompt para Wiki LLM (Second Brain)

## Versão recomendada (forte)

Agora você é o arquiteto e mantenedor da minha Wiki com LLM.

Use o texto acima como documento de especificação do sistema. Sua função é transformar essa ideia em uma implementação prática completa para Obsidian, em Markdown, pronta para uso imediato.

Implemente:
- `CLAUDE.md`
- `index.md`
- `log.md`
- estrutura de pastas
- convenções de escrita e links
- fluxo de ingestão
- fluxo de query
- fluxo de lint/manutenção
- primeiro exemplo de ingestão real

Importante:
- não apenas explique
- escreva os arquivos completos
- assuma que este será meu sistema oficial de segundo cérebro
- a partir de agora, todas as respostas devem seguir esse esquema

## Versão alternativa

Agora você é meu agente de Wiki com LLM.

Use o texto acima como especificação principal da arquitetura e implemente esse sistema como meu segundo cérebro completo.

Sua tarefa é:

1. Criar o arquivo `CLAUDE.md` com todas as regras operacionais do sistema
2. Criar o `index.md` inicial da wiki
3. Criar o `log.md` inicial
4. Definir a estrutura de pastas completa
5. Definir convenções de nomes, links internos, ingestão, manutenção e consulta
6. Mostrar um primeiro exemplo real de ingestão de conteúdo
7. Explicar passo a passo como vou operar esse sistema no dia a dia

Requisitos:
- Use Markdown em todos os arquivos
- Siga a lógica de wiki persistente descrita no texto-base
- Estruture tudo para uso em Obsidian
- Mantenha o sistema simples, legível e escalável
- Considere que esta wiki será meu “second brain” de longo prazo
- Toda interação futura deve respeitar esse esquema

Entrega esperada:
- conteúdo completo de `CLAUDE.md`
- conteúdo inicial de `index.md`
- conteúdo inicial de `log.md`
- árvore de pastas recomendada
- exemplo prático de ingestão
- instruções operacionais
